---
name: BM25(Okapi BM25)
category: 情報検索・ランキング
subcategory: スコアリング
complexity: O(検索語数)
summary: 単語の出現頻度と文書長を考慮してクエリとの関連度を採点する、検索エンジンの標準的なランキング関数。
---

## 概要

検索エンジンが「このクエリに対して、どの文書が最も関連性が高いか」を判定するための、確率的検索モデルに基づくランキング関数。TF-IDFの発展形にあたり、1970〜1990年代にかけて研究・改良が重ねられ、"BM"はBest Matchの略、25はそのバージョン番号。ElasticsearchやApache Solr、OpenSearchなど、現代の主要な全文検索エンジンで標準のスコアリング関数として採用されている、事実上の業界標準。

## 仕組み

BM25のスコアは、クエリに含まれる各単語について、以下の要素を組み合わせて計算される。

1. **単語の出現頻度(TF)**: その単語が文書中に多く出現するほどスコアが上がるが、**出現回数の増加とともに効果が徐々に飽和する**(TF-IDFの単純な線形加算とは異なり、10回目の出現は1回目ほどスコアに寄与しない、という現実的な調整が入っている)
2. **逆文書頻度(IDF)**: 多くの文書に出現するありふれた単語(「は」「の」など)は重要度が低いとみなし、珍しい単語ほど高いスコアを与える
3. **文書長の正規化**: 長い文書は単語が多く出現しやすいため、平均的な文書長と比較して**長い文書のスコアを適度に割り引く**補正が入っている(パラメータbで調整)

TF-IDFが単純な掛け算で構成されるのに対し、BM25は「出現頻度の飽和」と「文書長の正規化」という2つの現実的な調整パラメータ(k1とb)を導入することで、実務での検索精度を大きく向上させている。

## 特性・トレードオフ

- **計算量**: クエリに含まれる検索語ごとにスコアを計算するため、実質O(検索語数)(事前に構築された転置インデックスを使うことが前提)
- **TF-IDFとの違い**: TF-IDFよりも出現頻度・文書長の扱いが洗練されており、実務上の検索精度で優れることが広く確認されている。ただし両者とも「単語の意味」までは考慮しない、あくまで語彙の一致に基づくスコアリングである点は共通
- **ベクトル検索との対比**: BM25は「キーワードの一致度」に基づくスコアリング(スパース検索)であり、埋め込みベクトルによる「意味的な近さ」を測るベクトル検索(密検索)とは異なるアプローチ。近年は両者を組み合わせるハイブリッド検索(RRFなどで統合)が主流になっている
- **使いどころ**: Elasticsearch・Solr・OpenSearchなど主要な全文検索エンジンのデフォルトのランキング関数、ドキュメント検索・ECサイトの商品検索・社内wiki検索など、テキスト検索が必要なほぼ全てのシステム

## 実装例

```python
import math


def bm25_score(
    query: list[str],
    doc: list[str],
    corpus: list[list[str]],
    k1: float = 1.5,
    b: float = 0.75,
) -> float:
    n = len(corpus)
    avgdl = sum(len(d) for d in corpus) / n
    doc_len = len(doc)

    score = 0.0
    for term in query:
        df = sum(1 for d in corpus if term in d)
        if df == 0:
            continue
        idf = math.log((n - df + 0.5) / (df + 0.5) + 1)
        f = doc.count(term)
        numerator = f * (k1 + 1)
        denominator = f + k1 * (1 - b + b * doc_len / avgdl)
        score += idf * (numerator / denominator)
    return score
```

```typescript
function bm25Score(
  query: string[],
  doc: string[],
  corpus: string[][],
  k1 = 1.5,
  b = 0.75,
): number {
  const n = corpus.length;
  const avgdl = corpus.reduce((sum, d) => sum + d.length, 0) / n;
  const docLen = doc.length;

  let score = 0;
  for (const term of query) {
    const df = corpus.filter((d) => d.includes(term)).length;
    if (df === 0) continue;
    const idf = Math.log((n - df + 0.5) / (df + 0.5) + 1);
    const f = doc.filter((w) => w === term).length;
    const numerator = f * (k1 + 1);
    const denominator = f + k1 * (1 - b + (b * docLen) / avgdl);
    score += idf * (numerator / denominator);
  }
  return score;
}
```

```cpp
#include <vector>
#include <string>
#include <algorithm>
#include <cmath>

using Document = std::vector<std::string>;

double bm25Score(const std::vector<std::string>& query, const Document& doc,
                  const std::vector<Document>& corpus, double k1 = 1.5, double b = 0.75) {
    double n = static_cast<double>(corpus.size());
    double totalLen = 0;
    for (const auto& d : corpus) totalLen += d.size();
    double avgdl = totalLen / n;
    double docLen = static_cast<double>(doc.size());

    double score = 0.0;
    for (const auto& term : query) {
        int df = 0;
        for (const auto& d : corpus) {
            if (std::find(d.begin(), d.end(), term) != d.end()) df++;
        }
        if (df == 0) continue;
        double idf = std::log((n - df + 0.5) / (df + 0.5) + 1);
        int f = static_cast<int>(std::count(doc.begin(), doc.end(), term));
        double numerator = f * (k1 + 1);
        double denominator = f + k1 * (1 - b + b * docLen / avgdl);
        score += idf * (numerator / denominator);
    }
    return score;
}
```

```csharp
using System;

static double Bm25Score(List<string> query, List<string> doc, List<List<string>> corpus,
    double k1 = 1.5, double b = 0.75)
{
    int n = corpus.Count;
    double avgdl = corpus.Average(d => d.Count);
    int docLen = doc.Count;

    double score = 0;
    foreach (var term in query)

        double denominator = f + k1 * (1 - b + b * docLen / avgdl);

}
```
