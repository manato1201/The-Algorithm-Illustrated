## 概要

2つの整数の最大公約数(GCD)を求める、紀元前300年頃のユークリッドの著作『原論』にまで遡る、**現存する中で最も古いアルゴリズムのひとつ**。分数の約分から暗号理論に至るまで、数論のほぼ全ての分野で土台として使われ続けている。

## 仕組み

「aとbの最大公約数は、aをbで割った余りとbの最大公約数に等しい」という性質だけを使う。

1. 2つの整数a, b(a ≥ b)が与えられる
2. aをbで割った余りrを求める
3. rが0であれば、その時点のbが最大公約数
4. rが0でなければ、aをb、bをrに置き換えて2に戻る

例えば `gcd(48, 18)` を求めるなら、`48 = 18×2 + 12` → `gcd(18, 12)`、`18 = 12×1 + 6` → `gcd(12, 6)`、`12 = 6×2 + 0` → 答えは6、という具合に、驚くほど少ない反復で答えにたどり着く。

## 特性・トレードオフ

- **計算量**: O(log min(a,b))。連続する2つのフィボナッチ数を入力したときに反復回数が最大になることが知られており(ラメの定理)、これがこのアルゴリズムの最悪ケースの計算量の根拠になっている
- **驚異的な収束の速さ**: 2000年以上前に考案されたにもかかわらず、今なお最大公約数を求める実用上最速のアルゴリズムの一つであり続けている
- **拡張ユークリッドの互除法への発展**: 最大公約数だけでなく、`ax + by = gcd(a,b)` を満たす係数x, yまで同時に求める拡張版があり、これがモジュラ逆数の計算(暗号理論で頻出)に直結する
- **使いどころ**: 分数の約分、暗号理論(RSA暗号の鍵生成における互いに素の判定)、周期性の解析(歯車の噛み合わせ問題など)、あらゆるプログラミング言語の標準ライブラリに実装されている基礎中の基礎

## 実装例

```python
def gcd(a: int, b: int) -> int:
    while b != 0:
        a, b = b, a % b
    return a
```

```typescript
function gcd(a: number, b: number): number {
  while (b !== 0) {
    [a, b] = [b, a % b];
  }
  return a;
}
```

```cpp
long long gcd(long long a, long long b) {
    while (b != 0) {
        long long r = a % b;
        a = b;
        b = r;
    }
    return a;
}
```

```rust
fn gcd(mut a: u64, mut b: u64) -> u64 {
    while b != 0 {
        let r = a % b;
        a = b;
        b = r;
    }
    a
}
```

```csharp
static long Gcd(long a, long b)
{
    while (b != 0)
    {
        (a, b) = (b, a % b);
    }
    return a;
}
```
